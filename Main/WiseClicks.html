Project:

'Spot the Scam' 2D Game

Project Type
Group project
Project Goals
Bring awareness about the different types of scams
in today's world through a mobile game.
Project Task
To create a 2D game in Unity to bring awareness about online scams.
Tools
Unity
C#
Figma
Duration
8 months

OVERVIEW

Background

This project was part of my university work-study scheme that I joined and I was tasked to work on a 'Anti-Scam' game where users will identify if the given content is 'Real' or 'Scam'.
Given the small team with only 2 developers (including me), my position combined responsibilities as a game developer and UI/UX designer, where I handled both gameplay development and the design of the UI interface and visual assets, effectively working as a full-stack game developer within the project team.

WIREFRAMES

When I first started on this project, it was clear the current development of the app was still in early phase as it lacks user flows. Hence, I created low-fidelity mock-ups of how I interpreted the app and make the gameplay stand out first.

VISUAL RESEARCH

After receiving approval, I researched suitable UI styles for the app. Given the broad theme, I refined the direction toward a sci-fi theme to reflect the role of AI, where it drive innovation but enable more sophisticated scams. This visual style will help convey the app’s focus on modern, tech-driven challenges. I also wanted it to be very simple as our target audience is towards the elderly.

DEVELOPMENT PROCESS

After importing the designs into Unity, I ensured all UI assets were properly scaled across devices by setting the pivot point to support different mobile aspect ratios.

Next, I focused on gameplay scripting. I developed a reusable C# script to handle the core “Real vs Scam” gameplay mechanics. Although each level presents different content types (e.g., email, phone calls, SMS), the underlying interaction logic is the same.
By having this reusable script, it allowed consistent behavior across scenes and made the system easy to extend to new levels.
Below displays the script.



After the core gameplay mechanics were in place, I proposed adding an additional quiz component to deepen player engagement. This new section uses scenario-based questions to test users’ knowledge and awareness of scams.
Below is the script for the quiz system, designed with replayability in mind by generating a randomized set of questions in a different order each time the game is played.

FINAL RESULT

While the game still had room for improvement and a few features left to explore, the core gameplay was fully functional. This experience taught me a lot about building, refining, and shipping a real product for both IOS and android.
